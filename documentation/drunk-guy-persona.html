As a drunk user, me and the boys need to get a late night fourth meal without spending time arguing.

Use-Case and Interaction Flow

Use-Case

User SpaghettiYeti and his buddies need food at 3 am. They can never decide where to eat diplomatically, so they found a simple app that will help them decide quickly.

Interaction Flow

Precondition

User opens the Restaurant Tinder app.

Post-condition

The winning restaurant has been chosen.

  • RT opens to the home screen and provides 3 options of "create group", "join a group?", or "flying solo?"
  • Admin User taps "join a group".
  • RT brings up list of pre-made groups.
  • User selects group called "THE BOYYZ".
  • After everyone joins, RT asks admin user for area code.
  • Admin user types in area code and hits next button.
  • RT asks if Admin user wants delivery or dine-in.
  • Admin user taps delivery button
  • RT finds 9 restaurants in area that deliver and are open, and shows user the number and the first choice with an option of swipe right or left.
  • User swipes through all the choices
  • RT says "tallying votes..." while others are still voting.
  • once all users have voted, RT shows 5 restaurants remain and begins new round of voting.
  • User votes again
  • RT says "tallying votes..."
  • after votes are in, RT says there is a 3 way tie.
  • after a few moments, a randomizer initiates and shows the group the winner.
